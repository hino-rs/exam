# Git/GitHub

## 開発の流れ

### 1. プル

手元を最新の状態にする

1. examで右クリック
2. チームにカーソルを合わせて`プル(U)`をクリック

### 2. ブランチを切る

作業内容を決めてブランチを切る

1. examで右クリック
2. チームにカーソルを合わせて`切り替え(W)` → `新規ブランチ(N)`
3. ブランチ名(規則後述)を入力してブランチを切る(`新規ブランチをチェックアウトする` という項目にチェック)

ブランチ名は、以下の表に則って決めます。

|作業分類|ブランチ名|
|---|---|
|新機能の実装|`feat/○○`|
|バグなどの修正|`fix/○○`|
|リファクタリング|`refactor/○○`|
|ドキュメント関連|`docs/○○`|
|スタイリング(コードを整えるのもCSSも)|`style/○○`|
|雑用|`chore/○○`|
|テスト|`test/○○`|
|パフォーマンス改善|`perf/○○`|

例: ログインページの実装<br>
  → `feat/login-page` でブランチを切る
  
### 3. コーディングして保存

実際にコードを記述しコミット

1. キリの良いところまでコーディング
2. examで右クリック
3. チームにカーソルを合わせて`コミット`
4. `Gitステージング`の画面が出るので対象のファイルを`+`や`++`ボタンを教えてステージへ追加
5. コミットメッセージを入力
6. コミットボタンを押す

- 基本的に編集内容の分類で分ける(たとえば修正と実装を1コミットにしない。修正は修正、実装は実装)。

- コミットメッセージは自由。「追加」「修正」など編集の分類を分かりやすく。
	- 例: ログインページの実装
	  - `feat: add login page`
	  - `feat: ログインページの実装`
	  - `追加: ログインページの実装` など
	
	困ったら[Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/#summary)
  
### 4. リモートに送る

自分のPCにあるブランチをサーバーにアップロード

1. examで右クリック
2. チームにカーソルを合わせて`originへプッシュ(P)`

### 5. プルリクエストを送る

GItHubの当リポジトリページを開き、画面上部に出てくるプルリクエスト作成ボタンを押し、作成する。
メンバーのレビューが完了したらマージされる。

### 6. 不要になったブランチを消す

1. 2の操作と同じように、ブランチを`master`へ切り替える
2. 1の操作と同じように、リモートの最新状態を取り込むためにプルをする
3. 画面上部の`ウィンドウ`をクリック
4. `ビューの表示`から`その他`
5. `Git`の中の`Gitリポジトリー`をクリック
6. ビューがでるので、その中の`exam/ブランチ/ローカル/[作成したブランチ]`で右クリック
7. 削除

2の作業へ

## 用語

- リポジトリ: プロジェクト専用フォルダ
- ローカル: Git(自分のPC)
- リモート: GitHub
- フェッチ: リモートの最新状態を確認
- プル: リモートからローカルへ合流
- プッシュ: ローカルからリモートへ送る
- コミット: ローカルに保存
- ブランチ: 編集の分岐(一本化せず、Aという道と、Bという道で編集を分けられる)
- main/master: リポジトリのメインブランチ
- マージ: ブランチを合併させる。特にGitHub上ではmain/masterへ
- origin: "プロジェクト"を指してると考えればいい
- プルリクエスト: 自分の編集のマージを提案
- コンフリクト: 衝突 AさんとBさんの同じ箇所への編集で、mainに矛盾が起きること(多分ブランチ切りで防げる)
- チェックアウト: ブランチを切り替える
- HEAD: 今自分が作業しているブランチの最新の場所
- ステージ: 保存対象

## コンソールで直接操作すれば確実

原因がeclipseの不具合に感じたとき、PowerShellやcmdでgit操作ができます。

```bash
# 現在の変更状態を確認
git status

# 変更したファイル全てステージング
git add .

# 指定のファイルをステージング(例)
git add memo.txt

# コミット
git commit -m "コミットメッセージをここに書く"

# プル
git pull

# プッシュ
git push 

# フェッチ
git fetch

# ブランチ一覧表示
git branch

# 新しいブランチを作ってそこに切り替える
git checkout -b feat/login-page # など

# 別のブランチを今のブランチに取り込む
git merge feat/login-page # など

# ローカルブランチを消す
git branch -d feat/login-page # など
```

## 補足

- 作業の重複が起こらないためにも、プルはこまめに行うべきです。
- 例えばメモなどは、ファイル名をユニークな名前にしてもらえればブランチを切らなくても良いです。
- なぜブランチを切るかは、1. 競合を防ぐため 2. 変更の妥当性を評価するため 3. 履歴に残すため です。
- コミットはローカル、プッシュはリモートへの保存だと思ってください。
- Gitの関心対象から外すには`.gitignore`というファイルにファイル名や階層を書きます。
